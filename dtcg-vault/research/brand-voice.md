# Brand Voice — PointSav Linguistic Rules

This document defines the linguistic rules and brand voice for the PointSav Design System. These rules must be applied to all user-facing copy, documentation, and error messages.

## Voice Attributes

| Attribute | Description | Anti-pattern |
| :--- | :--- | :--- |
| **Confident** | Use active voice and declarative statements. | Apologetic or hesitant language. |
| **Direct** | Get to the point quickly. Minimize filler words. | Excessive jargon or corporate fluff. |
| **Professional** | Maintain a serious, expert tone. | Overly casual or "cute" messaging. |

## Tone Guidelines

1. **Clarity over cleverness:** Ensure the user understands the action or information immediately.
2. **Helpful but not verbose:** Provide the necessary information without overwhelming the user.
3. **Inclusive:** Use gender-neutral language and avoid idioms that don't translate well.

## Usage in CMS

The CMS platform will automatically audit research entries and component descriptions against these linguistic rules using the `project-slm` linguistic validator.
