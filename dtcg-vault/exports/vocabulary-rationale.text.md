# Design System Vocabulary

This TEXT artifact defines the naming conventions and linguistic rationale for the design system substrate.

## Naming Conventions

| Layer | Prefix | Example |
| :--- | :--- | :--- |
| **Primitives** | `ps-` | `--ps-color-neutral-10` |
| **Semantic** | `interactive-` | `--ps-interactive-primary` |
| **Components** | `comp-` | `--ps-btn-background` |

## Token Taxonomy

Tokens must follow the category-item-variant scale (e.g., `color.primary.50`). This ensures mathematical predictability for AI agents performing codegen.

## Doctrine Alignment

Linguistic choices follow Doctrine Claim #38, emphasizing PointSav-original vocabulary over hyperscaler imports.
